# Лабораторная работа №6. Обработка и валидация форм

## Инструкции по запуску

1. Скопируйте папку `lab6` в директорию `htdocs` XAMPP.
2. Запустите Apache.
3. Откройте `http://localhost/projects/usm/lab6/index.php`.

Проверка синтаксиса:

```bash
/Applications/XAMPP/xamppfiles/bin/php -l index.php
/Applications/XAMPP/xamppfiles/bin/php -l src/SecretNoteRepository.php
/Applications/XAMPP/xamppfiles/bin/php -l src/SecretNoteValidator.php
/Applications/XAMPP/xamppfiles/bin/php -l src/ValidatorInterface.php
```

## Описание работы

Выбрана тема проекта: мини-система тайных признаний. Реализована форма создания записи, серверная валидация, сохранение данных в JSON-файл и вывод сохраненных записей в таблице.

Модель данных содержит:

- `title` - короткий текстовый заголовок;
- `author` - автор;
- `category` - enum-значение из списка категорий;
- `mood` - enum-значение из списка настроений;
- `confession` - длинный текст;
- `created_at` - дата создания;
- `visibility` - checkbox со значением публичности;
- `updated_at` - дата последнего обновления, добавляется сервером.

## Краткая документация

- `index.php` отображает форму, принимает `POST`, вызывает валидатор и репозиторий, выводит таблицу записей.
- `ValidatorInterface` задает общий контракт валидаторов.
- `SecretNoteValidator` проверяет обязательные поля, длину текста, допустимые enum-значения и формат даты.
- `SecretNoteRepository` создает JSON-хранилище, добавляет записи и возвращает список с сортировкой по `created_at`, `category` или `author`.
- `data/notes.json` хранит записи в читаемом JSON-формате.
- HTML-форма использует стандартные атрибуты `required`, `minlength`, `maxlength` и `type="date"` для клиентской проверки перед отправкой.

## Примеры использования

1. Заполните форму.
2. Нажмите `Сохранить`.
3. При корректных данных запись появится в таблице.
4. Используйте ссылки сортировки над таблицей, чтобы упорядочить записи по дате, категории или автору.

## Безопасность

- Все данные перед выводом экранируются через `htmlspecialchars()`, чтобы снизить риск XSS.
- Серверная валидация не доверяет данным из `$_POST`.
- Значения `category`, `mood` и `visibility` проверяются по белому списку.

## Ответы на контрольные вопросы

1. Для отправки данных чаще всего используют `GET` и `POST`. HTML-форма напрямую поддерживает методы `GET` и `POST`.
2. Для доступа к данным формы используются суперглобальные переменные `$_GET`, `$_POST`, а также `$_REQUEST`, если нужно общее объединение данных запроса.
3. Безопасность обеспечивается серверной валидацией, фильтрацией допустимых значений, экранированием вывода через `htmlspecialchars()`, ограничением длины полей и отказом от доверия данным клиента.
